# NYXO Retro Gaming Station Redesign

## Design read

Redesign-overhaul of an existing Russian skin-marketplace landing. The page must behave like a modern physical gaming workstation: dense, tactile, operational and collectible, without copying Shopify BFCM objects, rooms, pinball composition or illustrations.

## Preserved contract

- Preserve the existing section order, anchor IDs, navigation labels and approved Russian copy.
- Preserve React, TypeScript, Vite, native CSS, mock product data and semantic FAQ disclosures.
- Preserve Alumni Sans for display headings and Golos Text for interface copy.
- Do not add generated images, external images, optional libraries or new landing sections.

## System language

- Treat the page as one connected machine chassis rather than isolated cards.
- Build major surfaces from matte plum plastic, painted dark metal, recessed warm displays, physical keys and precise status lamps.
- Use acid orange for actions and active selection. Use toxic yellow only for exact confirmation or focus.
- Every visible readout must contain real marketplace information: product name, game, price, condition, rarity or supported attribute.
- Use shallow bevels, inset highlights, controlled wear and one consistent warm light direction. No glass, bloom, glitch, scanlines, particles or RGB.

## Section design

### Hero

One integrated workstation. Copy and CTA live in the control deck; a large selected-product display occupies the visual centre; real product selectors and readouts update the screen. The first viewport must communicate NYXO, skins, marketplace action and retro gaming hardware without a solitary luxury object.

### Popular now

One inventory console with a selected-item screen and a bank of selectable inventory modules. Selection updates the featured item and exposes a clear active state. Slots retain name, image area, price, condition and supported attribute.

### How it works

One connected operation rail with four ordered states. The existing copy remains unchanged. Shared rails, state lamps and a final confirmation signal communicate one transaction instead of four cards.

### Inspection

A working inspection console with two accessible physical controls: full view and finish details. Switching changes the large media view while rarity, condition and attribute remain stable and readable.

### FAQ and footer

Quieter service modules within the same chassis. Native details remain. Footer forms the machine base rather than a separate generic link grid.

## Motion

- 120-240 ms causal transitions only.
- Physical controls depress by 1-2 px.
- Selected modules latch into place; screens change with opacity and short planar translation.
- No autonomous looping motion.
- Reduced-motion mode removes all nonessential transforms and smooth scrolling.

## Responsive behavior

- Desktop: integrated multi-module workstation with controlled overlap and depth.
- Tablet: primary screen stays dominant; controls and readouts reflow into two columns.
- Mobile: strict single column; no overlap; controls remain at least 44 px; all data and CTA labels remain visible.

## Acceptance

- Reads as a modern retro gaming station before it reads as a generic landing page.
- Remains practical for marketplace browsing, comparison and inspection.
- Contains no copied BFCM composition or decorative fake telemetry.
- Has no horizontal overflow at 320, 390, 430, 768, 1024 and 1440 px.
